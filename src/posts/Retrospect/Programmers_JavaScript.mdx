이번에는 5주 동안 진행했던 Programmers JavaScript Study 15기에 대해 정리해보려고 합니다. 
스터디를 하면서 미션을 통해 JavaScript를 깊이 있게 공부할 수 있었고 git의 pull request를 사용해 과제를 제출하는 과정도 익힐 수 있었습니다!

## 스터디 진행 방식

이 스터디에서는 매주 정해진 미션(필수 + 보너스)을 수행하고, 이를 pull request를 통해 제출해야 했습니다.
특히 pull request에는 구현 사항을 자세히 정리하고, 하단에 궁금한 점을 남겨서 다른 스터디원이나 멘토에게 코드 리뷰를 받을 수 있었습니다.

![출처: 2주차 미션 pull request ](/images/posts/Retrospect/Programmers_JavaScript/1.png)

서로의 코드를 보며 다양한 방식으로 문제를 해결하는 법을 배우는 것이 굉장히 유익했습니다

## 배운 점과 느낀 점

- **pull request 활용**
단순히 코드 제출이 아니라 구현 방식과 고민한 점을 정리해 남기면서 git을 더 효과적으로 활용하는 방법을 배울 수 있었습니다. 코드 리뷰를 통해 개선점을 찾고, 피드백을 반영하면서 코드 퀄리티도 높일 수 있었습니다.
-  **JavaScript 심화 학습**
단순한 문법 공부를 넘어 fetch API, localStorage, debounce 등 프로젝트에서 자주 활용되는 개념들을 직접 구현해볼 수 있어서 좋았습니다. 특히 어렵게 느껴졌던 비동기 처리 관련 개념을 확실하게 정리할 수 있었습니다.

- **코드 리뷰 문화 경험**
스터디원들의 다양한 코드 스타일을 보면서 새로운 접근 방식을 배울 수 있었고, 나의 코드에 대한 피드백을 통해 부족한 부분을 보완할 수 있었습니다. 직접 리뷰를 남겨보면서 다른 사람의 코드를 분석하는 연습도 되었습니다.





스터디가 끝나고 나니 혼자 공부할 때보다 훨씬 깊이 있고 다양한 코드 스타일을 경험해볼 수 있다는 점이 제일 좋게 다가왔습니다. JavaScript와 git을 더 잘 다룰 수 있게 되었고, 코드 리뷰를 통해 협업하는 법도 익힐 수 있었던 정말 좋은 경험이었습니다.

앞으로도 이런 방식으로 꾸준히 공부해 나가고 싶다는 생각이 들었습니다😊
